UI/UX Design Prompt for Aladin.AI Settings Page
Project Overview: Design the Settings page for Aladin.AI, a modern finance research assistant app inspired by Perplexity AI. This page allows users to customize their experience within the app, accessible via the sidebar under the Account section. The design must be responsive (mobile-first), default to dark mode (reflecting 12:20 AM PKT on October 12, 2025), and use gradients (#667eea to #764ba2) for accents, aligning with the app’s minimalist, elegant aesthetic (sans-serif fonts like Inter, subtle shadows, hover effects).
Target Users: Investors and traders (ages 25-50) seeking personalized app settings.
Core Principles:

Aesthetic: Clean, dark-themed (background #121212, text #ffffff), with a switch to light mode (background #f5f5f5, text #333333) via toggle. Use rounded cards (15px), gradient buttons, and smooth transitions.
UX Flow: Simple panel layout with distinct sections for theme, notifications, language, and API keys. Changes apply instantly or save on action.
Interactivity: Toggle for day/night mode, checkboxes, dropdowns, and a save button for API keys.
Accessibility & Performance: WCAG compliant, optimize for fast load (<2s).

Key Layout:

Settings Page:

Layout: Sidebar (20% width, collapsible on mobile), central content (60%) with a settings panel.
Sections:

Theme: Toggle switch labeled "Night Mode (Dark)" to "Day Mode (Light)". Default to dark at 12:20 AM PKT. Clicking toggles the entire app’s style (white to black or vice versa) with a sliding animation.
Notifications: Checkbox to enable/disable email alerts (e.g., market updates, payments).
Language: Dropdown with options (English default, Urdu, others).
API Keys: Input field for custom API keys (hidden unless admin), with a "Save API Key" button.


Interactions: Toggle updates UI instantly (persist via localStorage). Checkbox and dropdown changes save on selection. Save button validates API key (show error if invalid).
Mobile: Stacked sections, hamburger menu for sidebar.


Error States: Friendly message (e.g., "Invalid API key") with a retry option.

Components & Patterns:

Cards: Rounded (15px), shadow, hover lift for each section.
Buttons: Gradient (#667eea to #764ba2), white text, rounded (10px).
Toggle: Custom switch with sliding indicator (white dot on gradient track).
Animations: Fade-in on load, scale on hover.

Branding & Visuals:

Colors: Primary #764ba2, Accent #667eea, Dark #121212, Light #f5f5f5, Text (Dark) #ffffff, Text (Light) #333333.
Icons: SVG-based (e.g., gear for settings).
Typography: Headings bold (2xl), body (base).

Edge Cases: No admin access → hide API key field. Slow save → show spinner.
Deliverables: Integrate into app/settings/page.tsx in Next.js app router. Ensure compatibility with existing layout (e.g., /app/layout.tsx for theme class).
